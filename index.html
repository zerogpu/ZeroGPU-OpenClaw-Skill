Manual setup (if you prefer)

  1. Clone `ZeroGPU-OpenClaw-Plugin` and run `npm install`.
  2. Export `ZEROGPU_API_KEY` and `ZEROGPU_PROJECT_ID` in shell.
  3. Start plugin with `npm start` and verify `/health`.
  4. Add OpenClaw HTTP tool to `http://localhost:8787/v1/zerogpu/chat/completions`.
  5. Run one classification/summarization test and confirm savings metadata.

How ZeroGPU Routing works

01

Agent receives request

OpenClaw (or any agent) sends user intent to the routing tool.

02

Task is classified

Plugin detects whether task is summarization, classification, extraction, or follow-up generation.

03

Best small model selected

Model catalog is fetched and ranked dynamically with IAB-aware classification routing.

04

Response + savings returned

Tool returns normalized output plus per-call cost, estimated LLM cost, and savings.

Frequently asked questions

Can I call ZeroGPU directly without this tool?

Yes. The plugin is for routing, normalization, and savings tracking on top of direct API usage.

Do users need to host anything?

No. MVP setup is local clone + `npm start`. Hosting is optional for shared deployment.

How do I verify savings?

Use `/dashboard/events` and `/dashboard/summary`. Each response also includes `zerogpu` metadata.
